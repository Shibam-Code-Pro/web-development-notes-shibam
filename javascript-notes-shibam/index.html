1. Introduction to JavaScript

üìñ Explanation

JavaScript is a high-level, interpreted programming language that enables interactive web pages and is an essential part of web applications. Originally created to make web pages interactive, JavaScript has evolved into a versatile language used for both frontend and backend development.

JavaScript is dynamically typed, supports object-oriented and functional programming paradigms, and runs in browsers as well as server environments like Node.js.

üíª Code Example 

// Basic JavaScript example
console.log("Hello, JavaScript!");

// Variables and data types
let name = "John";
const age = 25;
var isStudent = true;

// Function
function greet(person) {
  return `Hello, ${person}!`;
}

// Calling the function
console.log(greet(name)); // Output: Hello, John!

// Object
const user = {
  name: "Alice",
  age: 30,
  sayHello() {
    console.log(`Hi, I'm ${this.name}`);
  }
};

user.sayHello(); // Output: Hi, I'm Alice

üîó Analogy

Think of JavaScript as the "brain" of a website. If HTML is the skeleton (structure) and CSS is the clothing (appearance), then JavaScript is the nervous system that makes everything interactive and responsive. Just like how your brain processes information and makes decisions, JavaScript processes user actions and makes the website respond accordingly.

  • ‚Ä¢ Semicolons are optional but recommended for clarity

  • ‚Ä¢ Variables declared with var have function scope, not block scope

2. Hello JavaScript

üìñ Explanation

The "Hello World" program is the traditional first program when learning any programming language. In JavaScript, we can output text to the console, display alerts, or modify HTML content. This chapter covers different ways to output "Hello World" and get started with JavaScript.

JavaScript can run in browsers (client-side) or on servers (Node.js). For beginners, we'll focus on browser-based JavaScript using the console and basic DOM manipulation.

üíª Code Example

  return `Hello, ${language}!`;

3. Comments

üìñ Explanation

Comments are text notes in your code that are ignored by the JavaScript engine. They're essential for documenting your code, explaining complex logic, and making your code more readable for yourself and other developers. Good comments explain the "why," not just the "what."

JavaScript supports two types of comments: single-line comments (//) and multi-line comments (/* */). Comments are also useful for temporarily disabling code during debugging.

4. Variables (var, let, const)

üìñ Explanation

Variables are containers that store data values. JavaScript has three ways to declare variables: var, let, and const. Each has different scoping rules and behaviors. Understanding these differences is crucial for writing reliable JavaScript code.

Modern JavaScript (ES6+) prefers let and const over var due to better scoping and fewer unexpected behaviors. Use const for values that won't change, let for values that will change, and avoid var in modern code.

üíª Code Example 

// var - function scoped, can be redeclared
var name = "John";
var name = "Jane"; // No error
console.log(name); // "Jane"

// let - block scoped, cannot be redeclared
let age = 25;
// let age = 30; // Error: Cannot redeclare
age = 30; // OK - can reassign
console.log(age); // 30

// const - block scoped, cannot be redeclared or reassigned
const PI = 3.14159;
// PI = 3.14; // Error: Cannot reassign
// const PI = 3.14; // Error: Cannot redeclare

// const with objects and arrays (contents can change)
const person = { name: "Alice", age: 28 };
person.age = 29; // OK - modifying property
person.city = "New York"; // OK - adding property
console.log(person); // { name: "Alice", age: 29, city: "New York" }

const colors = ["red", "green", "blue"];
colors.push("yellow"); // OK - modifying array
console.log(colors); // ["red", "green", "blue", "yellow"]

// Block scope demonstration
if (true) {
  let blockVar = "I'm in a block";
  const blockConst = "Me too";
  var functionVar = "I'm function scoped";
}

// console.log(blockVar); // Error: not defined
// console.log(blockConst); // Error: not defined
console.log(functionVar); // "I'm function scoped" - accessible!

üîó Analogy

Think of variables as labeled boxes in a warehouse. 'var' is like an old-style box that can be moved anywhere in the warehouse and relabeled. 'let' is like a modern box that stays in its designated section and can have its contents changed but not its label. 'const' is like a safety deposit box - once you put something in and label it, both the contents and label are permanent (though if it's a container, you can still organize what's inside).

‚ö†Ô∏è Gotchas

  • ‚Ä¢ var has function scope, not block scope (can cause unexpected behavior)
  • ‚Ä¢ var declarations are "hoisted" to the top of their scope
  • ‚Ä¢ const doesn't make objects/arrays immutable, just the reference
  • ‚Ä¢ Always declare variables before using them to avoid implicit globals

5. Data Types

üìñ Explanation

JavaScript has dynamic typing, meaning variables can hold different types of values. There are two main categories: primitive types (number, string, boolean, undefined, null, symbol, bigint) and non-primitive types (objects, arrays, functions). Understanding data types is crucial for writing effective JavaScript code.

JavaScript automatically converts between types when needed (type coercion), but this can sometimes lead to unexpected results. It's important to understand how different types behave and interact.

üíª Code Example 

// Primitive Data Types

// Number
let age = 25;
let price = 99.99;
let negative = -10;
let infinity = Infinity;
let notANumber = NaN;

// String
let name = "John";
let message = 'Hello World';
let template = `My name is ${name}`;

// Boolean
let isActive = true;
let isComplete = false;

// Undefined
let undefinedVar;
console.log(undefinedVar); // undefined

// Null
let emptyValue = null;

// Symbol (ES6+)
let symbol1 = Symbol('id');
let symbol2 = Symbol('id');
console.log(symbol1 === symbol2); // false

// BigInt (ES2020)
let bigNumber = 1234567890123456789012345678901234567890n;

// Non-Primitive Data Types

// Object
let person = {
  name: "Alice",
  age: 30,
  isStudent: false
};

// Array
let colors = ["red", "green", "blue"];
let numbers = [1, 2, 3, 4, 5];
let mixed = ["hello", 42, true, null];

// Function
function greet(name) {
  return `Hello, ${name}!`;
}

// Type checking
console.log(typeof age);        // "number"
console.log(typeof name);       // "string"
console.log(typeof isActive);   // "boolean"
console.log(typeof undefinedVar); // "undefined"
console.log(typeof emptyValue); // "object" (this is a known quirk!)
console.log(typeof person);     // "object"
console.log(typeof colors);     // "object"
console.log(typeof greet);      // "function"

// Type coercion examples
console.log("5" + 3);    // "53" (string concatenation)
console.log("5" - 3);    // 2 (numeric subtraction)
console.log("5" * "2");  // 10 (numeric multiplication)
console.log(true + 1);   // 2 (boolean to number)
console.log(false + 1);  // 1 (boolean to number)

üîó Analogy

Think of data types as different kinds of containers in a kitchen. Numbers are like measuring cups (precise quantities), strings are like labels (text information), booleans are like light switches (on/off states), objects are like recipe cards (collections of related information), and arrays are like spice racks (ordered lists of items). Just as you wouldn't measure flour with a label or turn on a measuring cup, each data type has its specific purpose and behavior.

‚ö†Ô∏è Gotchas

  • ‚Ä¢ typeof null returns "object" (historical JavaScript bug)
  • ‚Ä¢ NaN === NaN returns false (use Number.isNaN() instead)
  • ‚Ä¢ "5" + 3 = "53" but "5" - 3 = 2 (different coercion rules)
  • ‚Ä¢ Arrays and functions are objects (typeof array returns "object")

6. Operators

üìñ Explanation

Operators are symbols that perform operations on operands (values and variables). JavaScript has several types of operators: arithmetic, assignment, comparison, logical, and more. Understanding operators is essential for performing calculations, making decisions, and manipulating data.

JavaScript operators follow precedence rules (order of operations) and can have different behaviors with different data types due to type coercion.

üíª Code Example 

// Arithmetic Operators
let a = 10;
let b = 3;

console.log(a + b);  // 13 (addition)
console.log(a - b);  // 7 (subtraction)
console.log(a * b);  // 30 (multiplication)
console.log(a / b);  // 3.333... (division)
console.log(a % b);  // 1 (modulus/remainder)
console.log(a ** b); // 1000 (exponentiation)

// Assignment Operators
let x = 5;
x += 3;  // x = x + 3 = 8
x -= 2;  // x = x - 2 = 6
x *= 4;  // x = x * 4 = 24
x /= 3;  // x = x / 3 = 8
x %= 5;  // x = x % 5 = 3

// Increment/Decrement
let count = 0;
console.log(++count); // 1 (pre-increment)
console.log(count++); // 1 (post-increment, then count becomes 2)
console.log(--count); // 1 (pre-decrement)
console.log(count--); // 1 (post-decrement, then count becomes 0)

// Comparison Operators
console.log(5 == "5");   // true (loose equality)
console.log(5 === "5");  // false (strict equality)
console.log(5 != "6");   // true (loose inequality)
console.log(5 !== "5");  // true (strict inequality)
console.log(10 > 5);     // true
console.log(10 < 5);     // false
console.log(10 >= 10);   // true
console.log(10 <= 9);    // false

// Logical Operators
let isTrue = true;
let isFalse = false;

console.log(isTrue && isFalse);  // false (AND)
console.log(isTrue || isFalse);  // true (OR)
console.log(!isTrue);            // false (NOT)

// Short-circuit evaluation
let result = null || "default";  // "default"
let value = "hello" && "world";   // "world"

// Ternary Operator
let age = 18;
let status = age >= 18 ? "adult" : "minor";
console.log(status); // "adult"

// String Operators
let firstName = "John";
let lastName = "Doe";
let fullName = firstName + " " + lastName; // "John Doe"

// Type Operators
console.log(typeof 42);           // "number"
console.log("name" in {name: "John"}); // true
console.log([1,2,3] instanceof Array); // true

üîó Analogy

Think of operators as tools in a toolbox. Arithmetic operators (+, -, *, /) are like a calculator for doing math. Comparison operators (>, <, ===) are like a scale or ruler for measuring and comparing things. Logical operators (&&, ||, !) are like decision-making tools that help you combine conditions. Assignment operators (=, +=) are like labels that help you store and update values in containers.

‚ö†Ô∏è Gotchas

  • ‚Ä¢ Use === instead of == to avoid type coercion issues
  • ‚Ä¢ + operator concatenates strings but adds numbers
  • ‚Ä¢ Operator precedence: * and / before + and - (use parentheses for clarity)
  • ‚Ä¢ Pre-increment (++x) vs post-increment (x++) have different return values

7. Conditional Statements

üìñ Explanation

Conditional statements allow your program to make decisions and execute different code blocks based on specific conditions. JavaScript provides if, else if, else, and switch statements for controlling program flow. These are fundamental for creating dynamic and responsive applications.

Conditions are evaluated as boolean values (true or false). JavaScript uses truthy and falsy values, which means non-boolean values can also be used in conditions.

üíª Code Example 

// Basic if statement
let age = 18;
if (age >= 18) {
  console.log("You are an adult");
}

// if-else statement
let temperature = 25;
if (temperature > 30) {
  console.log("It's hot!");
} else {
  console.log("It's not too hot");
}

// if-else if-else chain
let score = 85;
if (score >= 90) {
  console.log("Grade: A");
} else if (score >= 80) {
  console.log("Grade: B");
} else if (score >= 70) {
  console.log("Grade: C");
} else if (score >= 60) {
  console.log("Grade: D");
} else {
  console.log("Grade: F");
}

// Nested conditionals
let weather = "sunny";
let hasUmbrella = false;

if (weather === "rainy") {
  if (hasUmbrella) {
    console.log("Go outside with umbrella");
  } else {
    console.log("Stay inside or get wet");
  }
} else {
  console.log("Enjoy the weather!");
}

// Switch statement
let day = "Monday";
switch (day) {
  case "Monday":
    console.log("Start of work week");
    break;
  case "Tuesday":
  case "Wednesday":
  case "Thursday":
    console.log("Midweek");
    break;
  case "Friday":
    console.log("TGIF!");
    break;
  case "Saturday":
  case "Sunday":
    console.log("Weekend!");
    break;
  default:
    console.log("Invalid day");
}

// Ternary operator (shorthand if-else)
let isLoggedIn = true;
let message = isLoggedIn ? "Welcome back!" : "Please log in";
console.log(message);

// Multiple conditions
let username = "john";
let password = "secret123";

if (username === "john" && password === "secret123") {
  console.log("Login successful");
} else {
  console.log("Invalid credentials");
}

// Truthy and falsy values
let value = "";
if (value) {
  console.log("Value is truthy");
} else {
  console.log("Value is falsy"); // This will execute
}

// Falsy values: false, 0, "", null, undefined, NaN
// Everything else is truthy

üîó Analogy

Think of conditional statements as a choose-your-own-adventure book. At each decision point, you check a condition ("Is the door locked?") and based on the answer, you follow different paths. If statements are like simple yes/no questions, while switch statements are like a multiple-choice menu where you pick one option from many possibilities.

‚ö†Ô∏è Gotchas

  • ‚Ä¢ Don't forget break statements in switch cases (causes fall-through)
  • ‚Ä¢ Use === for comparisons to avoid type coercion issues
  • ‚Ä¢ Remember falsy values: false, 0, "", null, undefined, NaN
  • ‚Ä¢ Avoid deeply nested if statements (consider early returns or switch)

8. Loops

üìñ Explanation

Loops allow you to execute code repeatedly based on a condition. JavaScript provides several types of loops: for, while, do-while, for...in, and for...of. Loops are essential for iterating through data, processing arrays, and automating repetitive tasks.

Understanding when to use each type of loop and how to control loop execution with break and continue statements is crucial for efficient programming.

üíª Code Example 

// For loop - most common
for (let i = 0; i < 5; i++) {
  console.log(`Iteration ${i}`);
}
// Output: Iteration 0, Iteration 1, ..., Iteration 4

// While loop - condition checked before execution
let count = 0;
while (count < 3) {
  console.log(`Count: ${count}`);
  count++;
}

// Do-while loop - condition checked after execution
let num = 0;
do {
  console.log(`Number: ${num}`);
  num++;
} while (num < 2);

// For...in loop - iterates over object properties
const person = { name: "Alice", age: 30, city: "New York" };
for (let key in person) {
  console.log(`${key}: ${person[key]}`);
}
// Output: name: Alice, age: 30, city: New York

// For...of loop - iterates over iterable values
const colors = ["red", "green", "blue"];
for (let color of colors) {
  console.log(color);
}
// Output: red, green, blue

// Nested loops
for (let i = 1; i <= 3; i++) {
  for (let j = 1; j <= 3; j++) {
    console.log(`${i} x ${j} = ${i * j}`);
  }
}

// Loop control statements
for (let i = 0; i < 10; i++) {
  if (i === 3) {
    continue; // Skip this iteration
  }
  if (i === 7) {
    break; // Exit the loop
  }
  console.log(i); // Prints: 0, 1, 2, 4, 5, 6
}

// Array iteration with forEach
const numbers = [1, 2, 3, 4, 5];
numbers.forEach((num, index) => {
  console.log(`Index ${index}: ${num}`);
});

// Infinite loop prevention
let attempts = 0;
while (true) {
  attempts++;
  if (attempts > 1000) {
    console.log("Breaking to prevent infinite loop");
    break;
  }
  // Some condition to break naturally
  if (Math.random() > 0.999) {
    console.log("Lucky break!");
    break;
  }
}

üîó Analogy

Think of loops as different types of repetitive tasks. A 'for' loop is like doing jumping jacks - you know exactly how many you'll do ("I'll do 20 jumping jacks"). A 'while' loop is like waiting for a bus - you keep waiting until it arrives. A 'for...of' loop is like going through your mail piece by piece, and a 'for...in' loop is like checking each pocket in your jacket to see what's inside.

‚ö†Ô∏è Gotchas

  • ‚Ä¢ Infinite loops can crash your browser (always have an exit condition)
  • ‚Ä¢ for...in iterates over all enumerable properties (including inherited ones)
  • ‚Ä¢ Modifying an array while iterating can cause unexpected behavior
  • ‚Ä¢ let vs var in loops: let creates block scope, var doesn't

16. DOM Events & Event Handling

üìñ Explanation

Events in JavaScript are actions that happen in the browser, such as clicking a button, typing in a form, or loading a page. JavaScript can "listen" for these events and execute code in response, making web pages interactive.

Event handling involves three parts: the event target (element), the event type (click, keypress, etc.), and the event handler (function that runs when the event occurs).

üíª Code Example 

// Basic event listener
const button = document.querySelector('#myButton');
button.addEventListener('click', function() {
    console.log('Button was clicked!');
});

// Event object
button.addEventListener('click', function(event) {
    console.log('Event type:', event.type);
    console.log('Target element:', event.target);
    console.log('Mouse coordinates:', event.clientX, event.clientY);
});

// Multiple event types
const input = document.querySelector('#textInput');
input.addEventListener('focus', () => console.log('Input focused'));
input.addEventListener('blur', () => console.log('Input lost focus'));
input.addEventListener('input', (e) => console.log('Value:', e.target.value));

// Keyboard events
document.addEventListener('keydown', function(event) {
    console.log('Key pressed:', event.key);
    if (event.key === 'Enter') {
        console.log('Enter key was pressed!');
    }
});

// Form events
const form = document.querySelector('form');
form.addEventListener('submit', function(event) {
    event.preventDefault(); // Prevent form submission
    console.log('Form submitted');
});

// Window events
window.addEventListener('load', () => {
    console.log('Page fully loaded');
});

// Removing event listeners
function handleClick() {
    console.log('This will be removed');
}
button.addEventListener('click', handleClick);
button.removeEventListener('click', handleClick);

üîî Analogy

Events are like doorbells in a house. When someone presses a doorbell (event), it rings (triggers), and you respond by going to the door (event handler). Different doorbells might make different sounds, and you can choose to ignore certain rings or respond differently based on who's at the door.

‚ö†Ô∏è Gotchas

  • ‚Ä¢ Event listeners on non-existent elements won't work

19. Async/Await & Promises Important

üìñ Explanation

Promises represent the eventual completion or failure of an asynchronous operation. They provide a cleaner alternative to callback functions for handling async code. Async/await is syntactic sugar built on top of promises, making asynchronous code look and behave more like synchronous code.

Promises have three states: pending (initial state), fulfilled (operation completed successfully), or rejected (operation failed). Async functions always return a promise, and await can only be used inside async functions.

üíª Code Example

  setTimeout(() => {
    const success = Math.random() > 0.5;
    if (success) {
      resolve("Operation successful!");
    } else {
      reject(new Error("Operation failed!"));
    }
  }, 1000);
});

// Using Promise with .then()/.catch()
myPromise
  .then(result => console.log(result))
  .catch(error => console.error(error));

// Async/Await syntax
async function handleAsyncOperation() {
  try {
    const result = await myPromise;
    console.log(result);
    return result;
  } catch (error) {
    console.error("Error:", error.message);
    throw error;
  }
}

// Promise chaining
function fetchUserData(userId) {
  return fetch(`/api/users/${userId}`)
    .then(response => response.json())
    .then(user => fetch(`/api/posts/${user.id}`))
    .then(response => response.json())
    .then(posts => ({ user, posts }));
}

// Equivalent with async/await
async function fetchUserDataAsync(userId) {
  const userResponse = await fetch(`/api/users/${userId}`);
  const user = await userResponse.json();
  
  const postsResponse = await fetch(`/api/posts/${user.id}`);
  const posts = await postsResponse.json();
  
  return { user, posts };
}

// Promise.all for parallel execution
async function fetchMultipleUsers(userIds) {
  const promises = userIds.map(id => fetch(`/api/users/${id}`));
  const responses = await Promise.all(promises);
  const users = await Promise.all(responses.map(r => r.json()));
  return users;
}

üéØ Analogy

Think of a Promise like ordering coffee at a busy caf√©. You place your order (create promise), get a receipt with a number (promise object), and can do other things while waiting (non-blocking). Eventually, your order is ready (resolved) or they run out of ingredients (rejected). Async/await is like having a personal assistant who waits in line for you and brings the coffee when it's ready.

  • ‚Ä¢ Forgetting to use 'await' makes the function return a Promise, not the value
  • ‚Ä¢ Await can only be used inside async functions or at the top level in modules

27. This Keyword

üìñ Explanation

The 'this' keyword refers to the object that is currently executing the function. Its value is determined by how a function is called, not where it's defined. Understanding 'this' is crucial for object-oriented programming in JavaScript.

The value of 'this' can change depending on the context: global scope, object method, constructor function, arrow functions, or explicit binding with call/apply/bind.

üíª Code Example 

// Global context
console.log(this); // Window object (browser)

// Object method
const person = {
  name: "Alice",
  greet() {
    console.log(this.name); // "Alice"
  }
};
person.greet();

// Constructor function
function Person(name) {
  this.name = name;
  this.greet = function() {
    console.log(this.name);
  };
}
const alice = new Person("Alice");
alice.greet(); // "Alice"

// Arrow functions (lexical this)
const obj = {
  name: "Bob",
  regularFunction() {
    console.log(this.name); // "Bob"
  },
  arrowFunction: () => {
    console.log(this.name); // undefined (global this)
  }
};

// Explicit binding
function sayHello() {
  console.log(`Hello, ${this.name}`);
}
const user = { name: "Charlie" };
sayHello.call(user); // "Hello, Charlie"
sayHello.apply(user); // "Hello, Charlie"
const boundFunction = sayHello.bind(user);
boundFunction(); // "Hello, Charlie"

üîó Analogy

Think of 'this' like a spotlight on a stage. The spotlight always points to whoever is currently performing (calling the function). If an actor (object) calls their own method, the spotlight points to them. If the method is called from somewhere else, the spotlight might point to a different performer or even the stage itself (global context).

‚ö†Ô∏è Gotchas

  • ‚Ä¢ Arrow functions don't have their own 'this' - they inherit it
  • ‚Ä¢ Event handlers change 'this' to the element that triggered the event
  • ‚Ä¢ Callback functions often lose their original 'this' context
  • ‚Ä¢ Strict mode changes 'this' behavior in global context

29. Performance Optimization

üìñ Explanation

Performance optimization involves making your JavaScript code run faster and use fewer resources. This includes optimizing algorithms, reducing DOM manipulation, managing memory efficiently, and using modern JavaScript features.

Key areas include minimizing reflows and repaints, debouncing/throttling events, lazy loading, code splitting, and using efficient data structures.

üíª Code Example 

// Debouncing function calls
function debounce(func, delay) {
  let timeoutId;
  return function(...args) {
    clearTimeout(timeoutId);
    timeoutId = setTimeout(() => func.apply(this, args), delay);
  };
}

const debouncedSearch = debounce(searchFunction, 300);
input.addEventListener('input', debouncedSearch);

// Efficient DOM manipulation
// Bad: Multiple DOM updates
for (let i = 0; i < 1000; i++) {
  document.body.appendChild(createElement());
}

// Good: Batch DOM updates
const fragment = document.createDocumentFragment();
for (let i = 0; i < 1000; i++) {
  fragment.appendChild(createElement());
}
document.body.appendChild(fragment);

// Use efficient array methods
const numbers = [1, 2, 3, 4, 5];
const doubled = numbers.map(n => n * 2);
const sum = numbers.reduce((acc, n) => acc + n, 0);

üìñ Explanation

üíª Mini Example

    });

// Service Worker (sw.js)
const CACHE_NAME = 'my-pwa-v1';
const urlsToCache = [
  '/',
  '/styles/main.css',
  '/scripts/main.js',
  '/offline.html'
];

// Install event - cache resources
self.addEventListener('install', event => {
  event.waitUntil(
    caches.open(CACHE_NAME)
      .then(cache => cache.addAll(urlsToCache))
  );
});

// Fetch event - serve from cache
self.addEventListener('fetch', event => {
  event.respondWith(
    caches.match(event.request)
      .then(response => {
        return response || fetch(event.request);
      })
  );
});

33. JavaScript Security (XSS Prevention, CSP)

üìñ Explanation

JavaScript security focuses on protecting web applications from common vulnerabilities like Cross-Site Scripting (XSS), injection attacks, and unauthorized access. Content Security Policy (CSP) is a security layer that helps detect and mitigate certain types of attacks by controlling which resources can be loaded and executed.

Proper security practices include input validation, output encoding, secure authentication, HTTPS usage, and implementing defense-in-depth strategies to protect both client-side and server-side code.

üíª Mini Example

  const div = document.createElement('div');
  div.textContent = input;
  return div.innerHTML;
}

// Safe DOM manipulation

  const sanitized = sanitizeInput(userInput);
  document.getElementById('content').textContent = sanitized; // Use textContent, not innerHTML
}

// CSP Header Example (server-side)
// Content-Security-Policy: default-src 'self'; script-src 'self' 'unsafe-inline'; style-src 'self' 'unsafe-inline'

// Secure data validation
function validateEmail(email) {
  const emailRegex = /^[^\s@]+@[^\s@]+\.[^\s@]+$/;
  return emailRegex.test(email) && email.length <= 254;
}

// Secure cookie handling
function setSecureCookie(name, value, days) {
  const expires = new Date();
  expires.setTime(expires.getTime() + (days * 24 * 60 * 60 * 1000));
  
  document.cookie = `${name}=${value}; expires=${expires.toUTCString()}; path=/; secure; samesite=strict`;
}

// CSRF Token handling
function addCSRFToken(formData) {
  const token = document.querySelector('meta[name="csrf-token"]').getAttribute('content');
  formData.append('_token', token);
  return formData;
}

  • ‚Ä¢ CSP can break legitimate functionality if configured too strictly
